# Recipe App Documentation

This folder contains all documentation for the Recipe App project, organized by category.

## 📁 **Documentation Structure**

### 🔧 **Setup Documentation** (`/setup/`)
- **`AI_ASSISTANT_SETUP.md`** - How to set up the AI assistant features
- **`API_KEYS_SETUP.md`** - Configuration for external API keys (OpenAI, ElevenLabs)
- **`ONBOARDING_DOCUMENTATION.md`** - User onboarding flow documentation

### 🚀 **Development Documentation** (`/development/`)
- **`COMPONENT_FOLDER_ORGANIZATION.md`** - Frontend component structure
- **`CSS_FOLDER_ORGANIZATION.md`** - CSS file organization and styling
- **`CSS_OPTIMIZATION_SUMMARY.md`** - CSS performance optimizations
- **`VOICE_FIRST_ARCHITECTURE.md`** - Voice-first design architecture

### 🐛 **Debug Documentation** (`/debug/`)
- **`debug_auth_issue.md`** - Authentication debugging notes
- **`debug_token_removal.md`** - Token removal debugging
- **`test_scraping_fixes.md`** - Web scraping fixes and improvements

## 📚 **Other Documentation**

### 📊 **Load Testing** (`/load_test_results/`)
- **`LOAD_TESTING_README.md`** - Load testing setup and usage
- **`load_test_report.html`** - Locust HTML report

### 🧪 **Performance Testing** (`/performance/`)
- **`README.md`** - Performance testing tools and usage
- **`PERFORMANCE_OPTIMIZATIONS.md`** - Complete optimization summary

## 🎯 **Quick Navigation**

- **New to the project?** → Start with `/setup/` documentation
- **Working on frontend?** → Check `/development/` for component and CSS guides
- **Debugging issues?** → Look in `/debug/` for troubleshooting notes
- **Testing performance?** → Use `/performance/` tools and `/load_test_results/` reports

## 📝 **Documentation Standards**

- All documentation should be in Markdown format
- Use clear, descriptive file names
- Include code examples where relevant
- Keep documentation up to date with code changes
